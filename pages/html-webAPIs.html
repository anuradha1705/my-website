HTML Web APIs

Web APIs (Application Programming Interfaces) allow your web pages to interact with the browser and device features beyond simple HTML content. These APIs provide powerful capabilities like accessing hardware, storing data, and communicating over networks.

üîç What are Web APIs?

Web APIs are sets of programming interfaces provided by browsers that enable you to do advanced tasks in web development, such as:

  • Accessing device hardware like camera, microphone, and geolocation.
  • Manipulating media and graphics with Canvas or WebGL.
  • Storing data offline using Web Storage and IndexedDB.
  • Communicating in real-time via WebSockets.
  • Interacting with service workers for offline capabilities.

This API lets you get the user's current location (with permission):

<button onclick="getLocation()">Get Location</button>
<p id="location"></p>

<script>
function getLocation() {
  if (navigator.geolocation) {
    navigator.geolocation.getCurrentPosition(showPosition);
  } else {
    document.getElementById("location").innerHTML = "Geolocation is not supported.";
  }
}

function showPosition(position) {
  document.getElementById("location").innerHTML =
    "Latitude: " + position.coords.latitude +
    ", Longitude: " + position.coords.longitude;
}
</script>

üìù Explanation

  • navigator.geolocation checks if geolocation is supported.
  • getCurrentPosition() asks the user for permission and returns location.
  • showPosition() displays the latitude and longitude.
  • Error handling is done in showError() function for better UX.
  • Web APIs require user permissions for privacy reasons.
