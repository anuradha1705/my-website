HTML Web Storage

Web Storage is a way to store data on the user's browser, allowing websites to save and retrieve data locally without sending it to a server.

üîç Types of Web Storage

  • Local Storage: Stores data with no expiration date. Data persists even after the browser is closed and reopened.
  • Session Storage: Stores data for the duration of the page session. Data is cleared when the page session ends (i.e., tab or browser closed).

‚öôÔ∏è Syntax

<!-- Storing data -->
localStorage.setItem('key', 'value');
sessionStorage.setItem('key', 'value');

<!-- Retrieving data -->
localStorage.getItem('key');
sessionStorage.getItem('key');

<!-- Removing data -->
localStorage.removeItem('key');
sessionStorage.removeItem('key');

<!-- Clearing all data -->
localStorage.clear();
sessionStorage.clear();

üìù Explanation

  • localStorage and sessionStorage provide simple key-value storage in the browser.
  • setItem(key, value) stores data with a specified key.
  • getItem(key) retrieves data for the given key.
  • removeItem(key) deletes data for the key.
  • clear() clears all storage items.
  • Local Storage data persists even after closing the browser, while Session Storage data clears when the tab/browser closes.
  • Data stored is always in string format. For objects, convert using JSON.stringify() and JSON.parse().

‚úÖ Summary

  • Web Storage allows saving data locally in the browser.
  • Local Storage persists indefinitely; Session Storage lasts for one session.
  • Data is stored as key-value pairs in string format.
  • Very useful for saving user preferences, login status, etc.
  • Simple API with methods: setItem, getItem, removeItem, and clear.
